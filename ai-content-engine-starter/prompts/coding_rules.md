Coding rules:
1. Always read tasks/agent_state.md before starting work.
2. Then read the corresponding task list file.
3. Execute only the current task.
4. Do not implement future tasks proactively.
5. Keep changes minimal and focused.
6. Update tasks/agent_state.md after completion.
7. Mark the completed task in the appropriate task list.
8. Append a short note to tasks/done_log.md.
9. If the task is too large, split it into a very small implementation that still advances the project safely.
10. Do not introduce new technologies unless explicitly requested.
11. Keep code idiomatic Go.
12. Prefer explicit code over clever abstractions.
13. Add tests when the task naturally requires them.
14. Do not implement more than one task per run.
15. If a task implies future work, only add placeholders, not full implementations of later tasks.
